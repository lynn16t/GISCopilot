核心设计理念 对话循环只负责理解意图、反问、收集参数、确认任务。用户确认后，LLM 输出 [TASK_CONFIRMED] 标记，OutputParser 检测到后触发完整分析流水线（embedding 检索 + LLM 工具选择 + TOML 文档 + GraphML）。
LLM 在对话阶段永远不输出 JSON 执行计划，Plan JSON 由分析流水线中的 tool_selection 步骤生成。

CONVERSATION_SYSTEM_PROMPT 的 4 种模式

Mode 1: TASK SUMMARY 信息足够 → 输出自然语言摘要 + 请求确认
「我将对 roads.shp 做500米缓冲区分析。确认后开始。」
Mode 2: TASK CONFIRMED 用户确认 → 输出 [TASK_CONFIRMED] + 精炼任务描述
「[TASK_CONFIRMED]\n对 roads.shp 做1000米缓冲区分析」
Mode 3: CLARIFYING QUESTION 信息不足 → 反问用户
「缓冲区距离是多少？」
Mode 4: CONVERSATIONAL REPLY 闲聊/知识问答 → 自然回复
「缓冲区分析是一种空间操作...」
LLM 在对话阶段永远不输出 JSON 执行计划。Plan JSON 由分析流水线的 tool_selection 步骤生成。

5 User "嗯但是改成1000米" — → 对话循环

→ ANALYZING
7 System _run_analysis() 执行 6 步流水线...
embedding 检索 → LLM 工具选择 → TOML 文档 → GraphML		 — ANALYZING
